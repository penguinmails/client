# Documentation Archive

This directory contains historical documentation that is no longer actively maintained but preserved for reference purposes. The archived content includes migration artifacts, temporary fixes, and outdated implementation guides.

## Archive Structure

### `/migration/`

Contains all documentation related to the Feature-Sliced Design (FSD) migration process that was completed. This includes:

- FSD migration strategies and guides
- Component migration summaries
- Violation resolution documentation
- Team migration guides
- Migration validation reports

### `/temporary/`

Reserved for temporary fixes and outdated guides that may be referenced for historical context but should not be used for current development.

## Important Notes

⚠️ **The documentation in this archive is not maintained and may contain outdated information.**

- Links within archived documents may be broken or point to non-existent files
- Code examples and implementation details may no longer reflect current practices
- For current documentation, please refer to the main `docs/` directory

## When to Reference Archive Content

Archive content should only be referenced when:

- Understanding historical decisions and their context
- Investigating legacy code patterns
- Researching past migration strategies for future reference
- Troubleshooting issues that may be related to historical changes
